# InversionWharf

Платформа дистрибуции коммерческих Docker-продуктов на серверы клиентов (on-prem).

Вендор ведёт каталог продуктов и версионированных релизов, регистрирует организации-клиенты и выдаёт права доступа. На сервере клиента работает **агент-демон**, который по команде организации устанавливает и обновляет продукты через Docker Engine и продлевает лицензионные лизы.

## Ключевые концепции

- **Pull-only**: агент инициирует все соединения; Control Plane не подключается к серверу клиента.
- **Инициатива у клиента**: установку/обновление запускает администратор организации вручную, не платформа.
- **Рантайм-лицензирование**: продукт проверяет лизенционный лиз в процессе работы; по истечении грейса (7 дней) — `HardStop`.
- **Целостность поставки**: образы фиксируются по digest, подписываются cosign; агент проверяет подпись до запуска.

## Быстрый старт (учебная версия)

> Полная инструкция: [`EDU_PROJECT_PLAN.md`](./EDU_PROJECT_PLAN.md)

```bash
cp .env.example .env          # задать пароли и секреты
docker compose up -d          # поднять всё окружение
open http://localhost:8080     # Operator Dashboard
open http://localhost:8095     # Agent LocalUI
```

Требования: Docker Engine ≥ 24, Docker Compose v2, 6 GB RAM.

## Архитектура (краткая схема)

```
Оператор ──► operator-web ──► api-gateway ──► auth / catalog / entitlement / license
                                                         │
Админ ──► agent (LocalUI) ──────────────────────────────┘
           └── Docker Engine (compose-проекты продуктов)
```

Подробная C4 L2 диаграмма — [`TECHNICAL_SPEC.md §2.2`](./TECHNICAL_SPEC.md).
