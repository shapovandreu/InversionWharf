Две независимые демо-версии веб-интерфейсов из [`TECHNICAL_SPEC.md`](../TECHNICAL_SPEC.md):

| Демо | Кто пользуется | Что показывает | Файл |
| --- | --- | --- | --- |
| **Control Plane** (Operator Dashboard) | Оператор вендора | Каталог продуктов/релизов, флот агентов, гранты доступа, **лицензии и льготный период**, телеметрия и аудит | [`control-plane/index.html`](./control-plane/index.html) |
| **Agent** (Local UI) | Админ организации-клиента | Статус апплайанса, доступные продукты, ручная установка/обновление, релиз-ноуты, здоровье установок, **рантайм-лицензия и грейс** | [`agent/index.html`](./agent/index.html) |

## Что можно потрогать (интерактив)

**Control Plane:**
- **Каталог** → открыть продукт → **+ Новый релиз** (semver, канал, релиз-ноуты) и **🚀 Канал** (продвижение stable/beta/edge).
- **Флот** → **+ Регистрация агента** (генерируется одноразовый enrollment-токен + ссылка на установщик), **+ Организация**.
- **Гранты** → выдать/отозвать грант доступа.
- **Лицензии** → отчёт **«Льготный период»** (организация · продукт · агент · осталось дней), продление/отзыв лиза, редактирование политики (TTL, грейс 7 дней, режим HardStop, maxInstances).
- На **Обзоре** — баннер с числом продуктов в льготном периоде; в **Телеметрии** — колонка состояния лицензии.

**Agent:**
- **Регистрация (enrollment, S1)** → экран первого запуска: ввод одноразового токена → анимация шагов (валидация токена → генерация ключа и CSR → `POST /v1/enroll` → подпись Vault PKI → robot-креды Harbor → mTLS-идентичность). Запустить можно кнопкой **↻ Перерегистрировать** во вкладке «Апплайанс».
- **Продукты** → ручная **Установка/Обновление/Удаление**, **Конфиг** (схема из каталога), **Патчноут**. При установке продукт получает лицензионный лиз.
- **Лицензия** → жизненный цикл лиза: **▶ Симулировать окончание лицензии** → льготный период 7 дней (продукт работает, идёт обратный отсчёт) → **⏭ Симулировать исчерпание грейса** → HardStop; **↻ Продлить лиз** возвращает в «Действует». Видны claims лиза и компоненты модуля (license-sidecar / License SDK / LicenseAgent).

## Запуск

Это статические single-file приложения на чистом HTML/CSS/JS (без сборки, без CDN,
моковые данные — бэкенд и интернет не нужны). Достаточно открыть HTML в браузере:

```bash
# просто открыть файл двойным кликом или из терминала
start demo/control-plane/index.html
start demo/agent/index.html

# при желании — локальный сервер
cd demo && python -m http.server 8080
#   http://localhost:8080/control-plane/
#   http://localhost:8080/agent/
```

> Это демо: все данные моковые и живут в памяти браузера. Никаких реальных вызовов
> к Control Plane, Harbor или Vault не выполняется. Цель — показать UX и
> информационную архитектуру обоих контуров (operator-gateway и локальный UI агента).
> Управление установкой/обновлением — только ручное (по решению организации).
